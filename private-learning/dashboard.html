Routing

Inbox para lanes

Toda saida do digest precisa cair em uma lane clara. Isso evita confundir nota pessoal, experimento e artefato que pode ser reaproveitado depois.

Lane 1

Deadline

Use quando existe prazo, resposta pendente ou compromisso. Se custa atraso, entra aqui.

Lane 2

Study Track

Use para conceito ou habilidade que precisa sair do nebuloso e virar capacidade real com `done` visivel.

Lane 3

Experiment

Use quando a pergunta ainda esta aberta. O objetivo e responder algo localmente, com artefato pequeno e reversivel.

Lane 4

Candidate

Use so depois de validar localmente. Aqui ficam os padroes que podem migrar um dia, nunca diretamente para o repo principal.
